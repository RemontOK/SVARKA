# Решение проблемы с прямыми переходами на GitHub Pages

## Проблема
При прямом переходе на страницу (например, `/SVARKA/catalog`) контент не загружается, нужно обновить страницу.

## Решение

### 1. Файл 404.html
GitHub Pages использует файл `404.html` для обработки несуществующих путей. Этот файл должен перенаправлять все запросы на `index.html`, чтобы React Router мог обработать маршрут.

### 2. Настройки GitHub Pages
Убедитесь, что в настройках репозитория (Settings → Pages):
- **Source**: GitHub Actions
- **Custom 404**: не включен (используется наш 404.html)

### 3. Проверка после деплоя
После деплоя проверьте:
1. Откройте сайт напрямую: `https://remontok.github.io/SVARKA/`
2. Перейдите на страницу каталога: `https://remontok.github.io/SVARKA/catalog`
3. Обновите страницу (F5) - контент должен загрузиться

### 4. Если проблема сохраняется
1. Очистите кэш браузера (Ctrl + Shift + Delete)
2. Проверьте консоль браузера на наличие ошибок
3. Убедитесь, что файл `404.html` скопирован в `dist/` при сборке
4. Проверьте логи в GitHub Actions

## Технические детали

- `404.html` перенаправляет все запросы на `index.html`
- React Router обрабатывает маршруты с учетом `basename="/SVARKA"`
- Все пути к ресурсам используют `import.meta.env.BASE_URL`
